### スマートエディタPlus 技術概要

#### 1. ツール概要
「スマートエディタPlus」は、Googleの生成AI (Gemini) を中核に据え、ビジネスにおける多様な知的生産活動を支援するために開発されたPython (Flask) 製のWebアプリケーションです。
主な機能は、高度な文章作成支援から、各種ドキュメント（PowerPoint、Word、Excel、レポート、作業要領書）の自動生成、非構造化データ（PDF、動画、音声）からの情報抽出、表データのバッチ処理、Microsoft 365環境とのシームレスな連携まで多岐にわたります。
Azure App Service上での運用を前提に設計されており、認証やデータ永続化にはAzureの各種サービスを効果的に活用しています。

---

#### 2. システム構成図（概念）
このツールは、以下のコンポーネントが連携して動作します。

*   **ユーザー (Webブラウザ)**
    *   `static/` と `templates/` にあるHTML/CSS/JSで構築されたUIを操作します。
*   **アプリケーションサーバー (Python/Flask on Azure App Service)**
    *   ユーザーからのHTTPリクエストやWebSocket通信を受け付けます。
    *   `routes.py` がリクエストを処理し、各種バックエンドロジックを呼び出す中核を担います。
    *   Gunicorn + Gevent により、多数の同時リクエストや、動画生成などの長時間処理を効率的に処理する非同期アーキテクチャを採用しています。
*   **Google Cloud Platform**
    *   **Vertex AI (Gemini 2.5):** 全てのAI機能（文章生成、要約、分析、コード生成、画像生成、動画の構成案・画像生成プロンプト・ナレーション原稿の作成など）の頭脳となります。
    *   **Speech-to-Text / Text-to-Speech:** リアルタイム文字起こしや、動画ナレーション生成で利用されます。
    *   **Cloud Storage:** アプリケーション共通のプロンプト定義や、ユーザーが作成したドキュメントレイアウト定義の保存場所です。
*   **Microsoft Azure**
    *   **App Service 認証 (Easy Auth):** Microsoft Entra IDと連携し、ユーザー認証を一元管理します。
    *   **Azure AI Search:** **(New)** RAG（検索拡張生成）機能において、社内ドキュメントやナレッジベースの高速なベクトル検索・キーワード検索を提供します。
    *   **Cosmos DB:** チャット履歴、AIの長期記憶(Memory)、非同期タスクの状況、Gemini APIの利用ログなどを構造化データとして永続化します。
    *   **Blob Storage:** 動画ファイルや生成されたドキュメントなど、処理中の一時的な大容量ファイルを保存します。
    *   **OpenAI Service:** GPTモデルを利用するためのAPIエンドポイントを提供します（セカンドオピニオン機能等で使用）。
*   **外部API / ツール**
    *   **Microsoft Graph API:** Microsoft 365連携（メール、ファイル、Teams、会議設定）で使用されます。
    *   **Iconify API:** AIが生成する図解内のアイコンデータを取得します。
    *   **FFmpeg:** サーバーサイドでの動画合成、音声結合、フォーマット変換を行う強力なコマンドラインツールとして利用されます。

#### 4. 機能一覧と技術的実装
各機能は、フロントエンドのJavaScriptがバックエンドのFlask APIを呼び出し、Flask側でGemini APIや各種ライブラリを利用して処理を実行する、という共通のアーキテクチャで構成されています。

| :--- | :--- | :--- |
| **多機能エディタ & AIチャット** | `index.html`, `script.js`, `routes.py`, `config.py`, `utils/gemini.py`, `utils/cosmos_db.py` | **多機能エディタ:** AIによる文章補完、校正、メール作成などを実装。`config.py`のプロンプトで制御。<br>**AIチャット:** 会話履歴をCosmos DBでツリー構造管理。**RAG機能**では`azure-search-documents`を用いて社内資料を検索し回答生成に利用。**会議設定**では会話から情報を抽出しGraph API連携用リンクを生成。**セカンドオピニオン**ではGeminiとAzure OpenAIを直列/並列で実行し回答を検証します。 |
| **ドキュメント比較** | `diff_checker.html`, `diff_checker.js`, `diff_checker.css`, `routes.py` | 2つのドキュメントファイル（Word, PDF等）をアップロードし、`file_processor.py`でテキストを抽出後、Gemini APIが意味的な差分を分析。結果はJSON形式で返され、フロントエンドで並べて表示されます。差分内容は`openpyxl`を用いてExcelファイルとしてエクスポートすることも可能です。 |
| **定型レポート生成** | `document_creator.html`, `document_creator.js`, `document_creator_styles.css`, `routes.py` | GUI上でテキストや画像のパーツを自由に配置し、各パーツの内容生成をAIに指示して定型的なプレゼンテーションを作成します。`interact.js`でパーツのドラッグ＆リサイズを実現し、レイアウト定義はJSONとして管理。バックエンドでは、このJSON定義と全体プロンプトに基づき、`python-pptx`ライブラリを使ってPowerPoint (.pptx) ファイルを動的に生成します。 |
| **データ抽出** | `extract_data_index.html`, `extract_data_script.js`, `routes.py` | PDFや動画ファイルから、ユーザーがプロンプトで指定した情報をGemini APIのマルチモーダル能力を活用して抽出します。PDFからの抽出では、サンプルPDFと期待されるJSON出力を例として提示するFew-shot learningに対応。動画処理は非同期タスクとして実行され、フロントエンドはタスクIDで進捗をポーリングします。抽出結果はJSON形式で表示され、ExcelやPDFとしてダウンロード可能です。また、`pdf2docx`ライブラリによるPDFのWord変換機能も提供します。 |
| **画像生成・編集** | `image_generator.html`, `image_generator.js`, `routes.py` | プロンプトに基づき、通常画像、バナー、アイコン、ストーリーボード、GIFアニメーションを生成・編集します。バックエンドではGeminiの画像生成モデルを呼び出します。バナー生成では、AIに背景を生成させた後、`Pillow`ライブラリでサーバーサイドでテキストを描画。ストーリーボードでは、まずAIに全シーンのプロンプトを計画させ、その後各画像を逐次生成します。GIFアニメーションの作成には`moviepy`を利用します。 |
| **要領書生成** | `manual_creator.html`, `manual_creator.js`, `routes.py` | 動画や参考資料から、画像付きの作業要領書（Word形式）を自動生成します。`opencv-python`で動画から鮮明なキーフレームを自動抽出し、手動キャプチャ画像や参考資料、動画の音声解析結果と共にGeminiに渡して手順をJSONで構造化。最後に、そのJSONと画像から`python-docx`を使ってWordファイルを生成します。 |
| **PowerPoint詳細生成** | `pptx_generator_detailed_index.html`, `routes.py`, `utils/ppt_gen.py` | **(Updated)** ユーザー指示やPDF資料から、AIがSVG形式でスライドをデザイン。`BeautifulSoup`でSVGを解析し、`cairosvg`で背景をPNG化、テキストを`python-pptx`オブジェクトとして再配置します。**動画エクスポート**機能では、マルチスピーカーTTSで音声を生成し、`moviepy`と`ffmpeg`を用いてスライド画像・音声・自動生成テロップを合成してMP4化します。 |
| **PowerPoint骨子生成** | `pptx_generator_index.html`, `pptx_generator_script.js`, `routes.py` | テキストからPowerPointの構成案（アウトライン）をJSON形式でAIに生成させます。フロントエンドでは`Sortable.js`を用いてGUIでスライドの順序変更や内容の編集が可能。編集後のJSONに基づき、バックエンドの`python-pptx`ライブラリが最終的な.pptxファイルを構築します。 |
| **スライド動画生成** | `slide_video_generator.html`, `slide_video_generator_script.js`, `routes.py` | アップロードされたPDFから`PyMuPDF(fitz)`で各ページを画像として抽出。抽出した画像と直前のスライドの文脈を元に、AI(Gemini)が各スライドのナレーション原稿を生成します。生成されたナレーションは`Google Text-to-Speech`で音声化され、最後に`moviepy`ライブラリを使って画像と音声を結合し、MP4形式の動画ファイルを生成します。この重い処理は非同期タスクとして実行されます。 |
| **リアルタイム文字起こし** | `recorder.html`, `recorder.js`, `smarteditorplus/realtime/handlers.py` | マイク音声を`Web Audio API`で取得し、`Flask-SocketIO`を通じてWebSocketでサーバーにストリーミング送信。サーバーサイドでは`gevent`キューで音声データを受け取り、Google Cloud Speech-to-Textのストリーミング認識APIに転送します。認識結果はクライアントに送り返され、メインエディタに自動で反映されます。 |
| **レポート生成** | `report.html`, `report.js`, `routes.py` | テキスト、画像、参考資料などを組み合わせて、Markdown, HTML, Word形式のレポートを生成します。バックエンドでは、複数ファイルを処理してプレースホルダーとBase64データのマップを作成。Geminiにプロンプトとファイル内容を渡し、指定形式のレポートを生成後、画像プレースホルダーをBase64データに置換して返します。 |
| **表データ処理** | `structured_data_index.html`, `structured_data_script.js`, `routes.py` | Excel/CSVファイルを`pandas`で読み込み、列情報をフロントエンドに返します。処理実行時、DataFrameを1行ずつループ処理し、プロンプト内のプレースホルダー (`{列名}`) をその行のデータで置換してGeminiを呼び出します。全行の処理後、結果を新しい列として追加し、ExcelまたはCSVとして出力します。 |
| **ドキュメント校正・翻訳** | `translate_index.html`, `translate_script.js`, `routes.py` | PowerPoint, Word, Excel, PDFファイルをアップロードし、AIによる校正や多言語翻訳（日・英・中）を非同期タスクとして実行します。各ファイル形式に対応したライブラリでテキストと書式情報を構造化データとして抽出し、チャンクに分割してGemini APIで並列処理。結果を元の構造に再適用し、書式を維持したまま新しいファイルを生成します。 |
| **動画編集** | `video_editor.html`, `video_editor.js`, `routes.py` | `moviepy`で動画のカット・結合、`librosa`と`noisereduce`でノイズ除去を行います。テロップ生成は、動画をチャンクに分割し、Geminiの音声認識APIで文字起こしとタイムスタンプを取得。AIによる不要シーンカットは、テロップ情報（発話区間）を元に重要度を判断し、`moviepy`で再構成します。すべての重い処理は非同期タスクとして実行されます。 |
| **動画パフォーマンス分析** | `video_analyzer.html`, `video_analyzer.js`, `routes.py` | 会議やプレゼンの動画をアップロードし、指定したシナリオに基づいてAIがパフォーマンスを評価・分析します。バックエンドは非同期タスクとして動画と補助的なトランスクリプトをGeminiのマルチモーダルAPIに送信。プロンプトで厳格なJSON形式の出力を指示し、各種スコアやフィードバックを生成させます。フロントエンドは、結果のJSONを`Chart.js`や動的HTMLで可視化し、インタラクティブなダッシュボードとして表示します。 |
| **動画解説 生成** | `video_explainer.html`, `video_explainer.js`, `routes.py` | ユーザーが入力したテキストや参考資料を元に、AI(Gemini)が動画全体の構成案、各シーンのナレーション原稿、および各シーンを表現する画像生成プロンプトを計画します。ナレーションはGoogle Text-to-Speech APIで音声(.mp3)に、画像プロンプトはGeminiの画像生成モデルで画像(.png)に変換。最後に、サーバーサイドで`moviepy`を使い、生成された画像と音声を結合して最終的な動画ファイル(.mp4)を生成します。この重い処理は非同期タスクとして実行されます。 |
| **動画ナレーション分析** | `video_narration_index.html`, `routes.py` | **(New)** 動画ファイルをアップロードし、その内容を「解説するスライド動画」に作り変えます。Geminiのマルチモーダル機能で映像と音声を解析し、要約脚本と説明用イラストを生成。これらを再構成して新しい解説動画(MP4)を生成します。 |

#### 5. 認証・認可
*   **認証方式:** Azure App Serviceの認証機能 (通称: Easy Auth) を利用。
*   **IDプロバイダー:** Microsoft Entra IDと連携。
*   **実装:**
*   `auth.json`: App Service上で認証が必要なパス (`/api/*`) を定義。
*   バックエンド (`routes.py`, `m365_search.py`): リクエストヘッダー (`X-MS-CLIENT-PRINCIPAL-ID`, `x-ms-token-aad-access-token`など) からユーザー情報やアクセストークンを取得し、Cosmos DBのパーティションキーやGraph APIの認証に使用。
*   フロントエンドでのMSAL (Microsoft Authentication Library) 等による複雑な認証処理は不要化されています。

---

#### 6. データ永続化
*   **Azure Cosmos DB (`utils/cosmos_db.py`):**
*   **チャット履歴:** ユーザーごとの会話セッション、メッセージ履歴、AIの長期記憶(Memory)を保存。
*   **非同期タスク:** 動画処理などの長時間タスクのID、状態、結果を保存。
*   **API利用ログ:** Gemini APIの呼び出し履歴（ユーザー、プロンプト、モデル等）を記録。
*   **Azure Blob Storage (`utils/storage.py`):**
*   動画生成タスクなどで生成された中間ファイル（PNG画像、音声ファイル）や最終成果物（MP4動画）を一時的に保存。
*   **Google Cloud Storage (`utils/storage.py`):**
*   アプリケーションで共通して使用するプロンプト定義や、ユーザーが作成したドキュメントレイアウト定義などを永続的に保存。

---

#### 7. 環境変数と設定
`.env` または実行環境変数に以下が必要です。

*   **Google Cloud:** `GOOGLE_APPLICATION_CREDENTIALS`, `GOOGLE_CLOUD_PROJECT` 等
*   **Azure:** `AZURE_STORAGE_CONNECTION_STRING`, `COSMOS_ENDPOINT`, `AZURE_OPENAI_ENDPOINT` 等
*   **Azure AI Search (New):** `AZURE_SEARCH_ENDPOINT`, `AZURE_SEARCH_KEY`, `AZURE_SEARCH_INDEX_NAME`
*   **AIモデル:** `CHAT_MODEL_NAME` (Gemini 1.5 Pro等), `IMAGE_GEN_MODEL` (Gemini 2.5等)
*   **外部ツール:** `FFMPEG_PATH` (動画生成機能で必須), `MARP_CLI_PATH` 等

---

#### 8. デプロイと実行
*   **実行環境:** Dockerコンテナ。
*   **Webサーバー:** Gunicorn + Gevent (非同期処理対応)。
*   **推奨:** Azure App Service for Containers。

---

#### 9. 運用・保守における注意点
*   **APIコスト:** Gemini 1.5 Proや画像生成モデル、Azure AI Searchの利用量に注意が必要です。
*   **非同期タスク:** 動画生成はサーバーリソース（CPU/メモリ）を消費するため、App Serviceのプラン選定に注意してください。
*   **依存関係:** `ffmpeg` などのバイナリツールがコンテナに含まれていることを確認してください。

### スマートエディタPlus「Insight Canvas」の機能概要

「Insight Canvas」は、データ分析の専門知識がないユーザーでも、AIとの対話を通じてデータからビジネスインサイトを発見し、具体的なアクションプランまで導き出すための統合分析プラットフォームです。

ExcelやCSVファイルをアップロードし、分析したい目的を自然言語で入力するだけで、**データの準備（プロファイリング）、分析の実行、結果の解釈、そして最終的なレポート作成まで、データ分析の全工程をAIが自動で実行**します。ユーザーは、AIが生成したグラフや考察を眺め、対話を通じてさらに深掘りするだけで、データに基づいた意思決定が可能になります。

#### 主な機能とプロセス

「Insight Canvas」は、大きく分けて4つのステップで分析プロジェクトを進行します。

##### 1. セットアップ支援：AIとの対話による分析設計
データ分析の最初の壁である「何から手をつければ良いか」をAIが解決します。

*   **データと目的の入力:** ユーザーは分析したいデータ（Excel/CSV）をアップロードし、「売上が落ちている原因を特定したい」といった分析の目的を文章で入力します。
*   **AIによるセットアップ提案:** AIがデータの中身（列名、データ型、サンプル値）と分析目的を理解し、以下を自動で提案します。
    *   **最適な分析レシピ:** 「顧客満足度分析」「品質不良データ分析」など、目的に合った分析手法を提案します。
    *   **列の役割推定:** 各列が「予測ターゲット」「日付」「売上金額」など、分析においてどのような役割を持つかを自動で推定します。
    *   **分析観点の提案:** 「特に注目すべき重要指標」や「参考資料の活用方針」など、分析を成功させるためのヒントを提示します。
*   **ユーザーによる確定:** ユーザーはAIの提案を確認し、必要に応じて列の役割などを修正して、分析プロジェクトを開始します。

##### 2. データプロファイリング：AIによるデータ品質診断
AIがデータの「健康診断」を行い、分析に適した状態に整えるための準備をします。

*   **データの自動評価:** AIがデータの基本統計量、欠損値の割合、値の分布などを自動で計算・可視化します。
*   **品質アラートと修正提案:** データの問題点（例：「『年齢』列に異常な値があります」）を日本語で指摘し、それを修正するための**Pythonコードまで自動で生成**します。ユーザーはワンクリックで修正を適用できます。
*   **特徴量エンジニアリング:** AIが「日付データから曜日や週末フラグを生成する」「地域と商品を組み合わせて新しい分析軸を作る」といった、より深い分析のための**新しい特徴量（列）の生成**を提案・実行します。

##### 3. 自動分析オーケストレーション：AIによる分析の計画・実行・解釈
この機能の核心部分です。AIがデータサイエンティストチームのように振る舞い、自律的に分析を進めます。

*   **分析計画の自動立案:** AIがプロジェクトの目的に基づき、「どの変数とどの変数を比較するか」「どのようなグラフで可視化すべきか」といった**具体的な分析計画を複数立案**します。
*   **コード生成と並列実行:** AIが各分析計画を実行するためのPythonコード（`pandas`, `matplotlib`, `scikit-learn`等を使用）を自動生成し、安全な環境で**並列実行**します。
*   **インサイトの自動解釈:** AIは、自身が生成したグラフや統計検定の結果を読み解き、「**このグラフから〇〇という傾向が見られます。これは△△が原因である可能性が示唆されます**」といった**日本語での解説文**を各分析結果に自動で付与します。
*   **高度な分析モデルの自動構築:**
    *   **予測モデル:** 「売上」や「解約率」などの予測ターゲットが設定されている場合、AIが自動で**予測モデル（回帰/分類）**を構築し、予測に最も影響を与えている**主要因**を特定します。
    *   **時系列予測:** 日付データが含まれている場合、AIが自動で**将来予測モデル**を構築し、今後のトレンドを予測します。
    *   **クラスター分析:** AIがデータを自動でいくつかのグループ（例：「優良顧客層」「離反予備軍」）に分類し、それぞれのグループに**特徴的な名前と解説**を付けます。

##### 4. 統合レポートとモデル活用：分析結果の資産化
分析から得られたインサイトを、すぐに使える形にまとめ、将来の分析に活かします。

*   **総括レポートの自動生成:** AIが全ての分析結果を統合し、**「エグゼクティブサマリー」「分析のストーリー」「推奨される次のアクション」**の3部構成からなる、経営層にも伝わる**最終レポート**を自動で生成します。
*   **多様なエクスポート:** 生成されたインサイトカードや総括レポートは、ワンクリックで**PowerPoint形式**やPDF形式でエクスポートできます。
*   **モデルレジストリ:** 作成された予測モデルは自動で保存・管理されます。過去に作成したモデルを呼び出し、「もし広告費を10%増やしたら売上はどうなる？」といった**What-ifシミュレーション**を実行したり、新しいデータでモデルを**再学習**させたりすることが可能です。

#### この機能の利点

*   **専門知識不要で高度な分析を実現:** データサイエンティストがいなくても、ビジネス課題から出発して、データに基づいた具体的なアクションプランまでを導き出せます。

*   **AIとの協調による高品質な意思決定:** AIの提案を人間が確認・修正し、対話を通じてさらに分析を深掘りすることで、AIの計算能力と人間のビジネス知見を融合させた、質の高い意思決定を支援します。
*   **分析ノウハウの資産化:** 実行された分析プロセスや作成された予測モデルはすべて保存・管理されるため、分析の再現性を担保し、組織全体のデータ活用能力を向上させます。

### 🧩 ソフトウェア設計支援機能

#### 1. プロダクト概要
ユーザーの「こんなアプリを作りたい」という自然言語のアイデアを入力とするだけで、要件定義、基本設計、詳細設計、そしてUIプロトタイプまでを一気通貫で生成・管理するAIアシスタント機能。

#### 2. ターゲットユーザーとユースケース
*   **Level 1 (ビジネスオーナー/企画者):**
    *   技術的な知識がなくても、対話形式でアイデアを具現化したい。
    *   投資家や開発会社に見せるための「コンセプトシート」や「動くモックアップ」が欲しい。
*   **Level 2 (エンジニア/アーキテクト):**
    *   厳密なドキュメント（ER図、シーケンス図、API仕様書）を効率的に作成したい。
    *   ドキュメント間の整合性を保ちながら、設計変更を自動反映させたい。

#### 3. コア機能とワークフロー

この機能は、ユーザーの習熟度に合わせて2つのモードを提供しています。

| モード | UIスタイル | 主な機能・生成物 |
| :--- | :--- | :--- |
| **Level 1: コンセプト具現化** | **ウィザード形式** | ・**コンセプトシート**: 背景、目的、スコープの定義<br>・**インフォグラフィック**: コンセプトを視覚化した画像生成<br>・**HTMLプロトタイプ**: 実際にブラウザで動く画面モックアップ<br>・**タスク分解**: 実装までのロードマップ作成 |
| **Level 2: 詳細設計エディタ** | **IDE (統合開発環境) 形式** | ・**ドキュメントツリー**: 要件定義～テスト計画まで約30種類の設計書<br>・**高度な図解生成**: PlantUML (UML/C4), Graphviz, Markmap<br>・**一括自動生成**: 依存関係を考慮し、全ドキュメントをバックグラウンド生成<br>・**チャット編集**: AIとの対話によるドキュメントの局部修正 |

#### 4. 生成されるドキュメント体系 (Vモデル対応)
`design_ai.py` の定義に基づき、以下のドキュメントが相互に依存関係を持ちながら生成されます。

*   **00. 計画:** プロジェクト概要、スコープ、課題解決、用語集
*   **10. 要件:** 概念DFD、ビジネスルール、状態遷移図、**高忠実度画面プロトタイプ(HTML/JS)**、機能一覧
*   **20. アーキテクチャ:** **C4モデル**（コンテキスト/コンテナ/コンポーネント）、インフラ構成図
*   **30. 詳細設計:** **ER図(DB設計)**、API仕様、シーケンス図、実装クラス図、バッチ設計
*   **40. テスト・移行:** テスト戦略、移行方針


#### 5. 特筆すべき機能 (Wow Factors)

1.  **コンテキスト認識型生成 (Dependency Injection):**
    *   ドキュメント間に依存関係（`dependencies`）が定義されており、例えば「詳細設計」を生成する際は、既に生成された「要件定義」や「アーキテクチャ図」の内容をAIが読み込んでから生成を行います。これにより、**矛盾のない設計書**が作成されます。

2.  **ハイブリッド・ストレージ戦略:**
    *   検索や一覧表示に必要な「メタデータ」は高速な **Cosmos DB** に、容量が大きくなる「ドキュメント本文・図解コード・チャット履歴」は安価な **Blob Storage** に保存するハイブリッド構成を採用しています。

3.  **マルチモーダルな出力:**
    *   テキストだけでなく、**PlantUML/Graphviz/Markmap** のコードを生成し、サーバーサイドまたはクライアントサイドでSVG描画します。
    *   さらに、Geminiの画像生成機能を利用して、プロジェクトのコンセプトを表す**インフォグラフィック画像**も自動生成します。

4.  **Word/Markdownエクスポート:**
    *   生成された設計書は、図解（SVGをPNG変換）やインフォグラフィックを含めた状態で、整形された **Wordファイル(.docx)** として一括ダウンロード可能です。

この機能追加により、スマートエディタPlusは単なるエディタから、**「上流工程から実装手前までをカバーするAI開発パートナー」**へと進化しています。

### スマートエディタPlus「Tool Studio」機能概要

「Tool Studio」は、**自然言語で指示するだけで、Microsoft 365と連携した高度な業務アプリケーションを即座に生成・実行・共有できる**、AI駆動型のノーコード/ローコード開発プラットフォームです。

最新版では、**ブラウザ上で動作するPython（Pyodide）実行環境**に加え、生成されたツールの品質をAIが自らテストする**「自己検証機能」**や、ツールの仕様についてAIに尋ねられる**「テクニカルアドバイザー機能」**を搭載しました。これにより、単なる自動生成を超えた、持続可能で信頼性の高いアプリ構築が可能になりました。

#### 主な機能とプロセス

「Tool Studio」は、以下の4つのステップで、アイデアを実用的なツールへと変換します。

##### 1. コンテキスト・ウィザード：インテリジェントなリソース特定
プログラミング知識がなくても、ツールが操作すべき対象をGUIで指定、あるいはAIが自動解決します。

*   **広範なM365連携:** SharePoint (サイト/リスト/ドライブ), OneDrive, Outlook (メールフォルダ), Planner (プラン), Teams (チーム/チャネル) を網羅。Outlookフォルダは階層構造を再帰的に取得し、選択可能です。
*   **ドリルダウン & 名前解決:** ウィザード形式での指定に加え、「請求書フォルダ」「営業部のプラン」といった名称から、AIとロジックが実在するリソースID（Site ID, Drive ID等）を自動的に特定・解決します。
*   **セキュアなコンテキスト注入:** 特定されたリソースIDや認証トークンは、セキュアな形で生成コードの実行環境（コンテキスト）に自動注入されます。

##### 2. ハイブリッド・ジェネレーティブ・コーディング & 自己検証
ユーザーの自然言語プロンプトに基づき、AIが最適な技術スタックを選択してコードを生成します。さらに、生成結果をAI自身がレビューします。

*   **UI & 制御 (HTML/JS):** モダンなUI（Tailwind CSS）と、Microsoft Graph APIを通じたファイル操作やメール送信などの制御ロジックをJavaScriptで生成します。
*   **高度データ処理 (Python on Browser):** 集計、分析、ファイル加工といった複雑な処理が必要な場合、AIは自動的に**Pythonコード**を生成します。
    *   **利用可能ライブラリ:** `pandas`, `openpyxl`, `Pillow`, `scikit-learn` 等に加え、`micropip` を用いて必要なパッケージを動的にインストールします。
    *   **役割分担:** 動画処理などはJS（ffmpeg.wasm）、データ加工はPythonといった最適な役割分担をAIが判断します。
*   **AI自己検証・修正 (Auto-Check):** 生成直後に、AIが「要件の網羅性」「UI/Logicの整合性」を自律的にチェックします。不備があれば即座に自己修正を行い、エラーのない高品質なコードを提供します。

##### 3. ランタイム & ライブエディタ：即時実行と対話的支援
生成されたツールは、サーバーへのデプロイを待つことなく、ブラウザ上で即座に動作します。

*   **WebAssemblyによるPython実行:** 生成されたPythonコードは、Pyodide技術により**ユーザーのブラウザ内（WebAssembly）**で安全に実行されます。サーバーにデータを送信することなく、ローカル環境のように高速にデータを処理します。
*   **ツールへの質問 (QA機能):** 実行中のツールについて「このボタンは何をするの？」「CSVのフォーマットは？」とチャットで質問すると、AIがソースコードと仕様書を解析し、テクニカルアドバイザーとして即座に回答します。
*   **対話的な修正 (Update):** 「グラフの色を変えて」「処理ロジックを修正して」といった指示で、UIとPythonロジックの両方をAIが修正・再生成します。実行時エラーが発生した場合も、エラーログをワンクリックでAIに送信し、自動修正させることが可能です。

##### 4. ソーシャル・エコシステム：組織知の蓄積と再利用
作成したツールは組織の資産として管理・共有され、改善のサイクルを生み出します。

*   **Discovery & エンゲージメント:** ツールはカテゴリ分類され、社内ライブラリに公開されます。「いいね（Like）」や「お気に入り（Favorite）」、閲覧数（View Count）による評価システムを備え、人気ツールが可視化されます。
*   **高度なリミックス (Fork & Context Switch):** 他者が作成した有用なツールを複製（Fork）する際、**「接続先（コンテキスト）を選び直す」**ことが可能です。
    *   例：「A支店の売上集計ツール」をコピーし、ロジックはそのままに「B支店のSharePointサイト」に接続先を切り替えて別ツールとして保存できます。

#### この機能の利点

*   **「高度な処理」の民主化:** Pythonの知識がなくても、ExcelのマクロやVBAを超える高度なデータ処理ツールを、現場担当者が自然言語だけで作成できます。
*   **信頼性の向上:** AIによる自己検証（Auto-Check）機能により、生成されるコードの品質が向上し、手戻りが減少します。
*   **セキュアで高速な実行:** データ処理をブラウザ（クライアントサイド）で完結させるため、機密データを外部サーバーに預けることなく、かつ高速に処理可能です。
*   **業務改善の高速化:** 「欲しい」と思った瞬間にアプリ化し、QA機能で仕様を確認しながら、その場で完成形に近づけることができます。
